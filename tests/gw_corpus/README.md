# gw_corpus — Greaseweazle Differential-Testing Corpus

Versioned collection of **inputs** and **frozen Greaseweazle outputs**
used by the differential conformance suite (`tests/conformance/`).

Strategy: [`docs/TESTER_STRATEGY.md`](../../docs/TESTER_STRATEGY.md) §2.3.

> **Status: SKELETON (P3.1, Welle 1).** Only the directory structure,
> tooling wiring, and a single *synthetic* input fixture exist. Real
> captures and frozen `gw` outputs land in Welle 2 (P3.2).

## Layout

```
tests/gw_corpus/
├── inputs/                       # what goes IN — frozen, never regenerated by tests
│   ├── flux_streams/             # SCP / HFE / KryoFlux RAW captures
│   │   ├── synthetic_mfm_dd_10trk.scp   # SKELETON placeholder (synthetic, not a real disk)
│   │   ├── gen_skeleton_fixture.py      # deterministic generator for the placeholder
│   │   └── MANIFEST.sha256             # sha256 of every fixture in this dir
│   ├── disk_images/              # IMG / ADF / IMD of known disks   (empty in skeleton)
│   └── mock_hardware_traces/     # recorded USB traces for HW sims  (empty in skeleton)
└── outputs_gw/                   # what `gw` makes of the inputs — frozen per gw version
    └── v1.23/                    # gw 1.23 reference outputs        (empty in skeleton)
```

## Why freeze inputs, not outputs

`gw` changes its output format between versions. If tests compared
against a live `gw` install they would be brittle. Instead:

- **inputs/** are frozen real captures — never change.
- **outputs_gw/v1.23/** is `gw` v1.23 run once over those inputs,
  committed as `*.bin` + `*.sha256`. CI needs no `gw` install.
- A `gw` version bump produces a new `outputs_gw/vX.Y/` directory; the
  diff against the previous one shows exactly what `gw` changed.

## The skeleton fixture is synthetic — and named honestly

`synthetic_mfm_dd_10trk.scp` is **not** a real disk. It is a
deterministic synthetic MFM flux stream (see `gen_skeleton_fixture.py`)
whose only purpose is to give the harness a non-empty input while the
real corpus is recorded.

Per DESIGN_PRINCIPLES ("Keine erfundenen Daten") it is deliberately
**not** named `ibm_360k_dos_boot.scp` or similar — a name that claims
real disk content the file does not have would be a forensic-honesty
violation. Welle-2 captures carry honest, content-accurate names.

## Maintaining the corpus

| Action | How |
|--------|-----|
| Add an input | drop the file in the right `inputs/` subdir, regenerate `MANIFEST.sha256` |
| Add gw reference output | `gw <cmd> <input>` once, commit `outputs_gw/v1.23/<input>.<cmd>.{bin,sha256}` |
| New gw version | create `outputs_gw/v<new>/`, re-run, commit; keep the old dir for cross-check |
| Size management | once inputs exceed a few MB total, move binaries to Git-LFS or `Axel051171/uft-test-corpus` |

## Regenerating the skeleton fixture

```bash
python3 tests/gw_corpus/inputs/flux_streams/gen_skeleton_fixture.py
sha256sum -c tests/gw_corpus/inputs/flux_streams/MANIFEST.sha256
```

Output is byte-stable across runs (fixed RNG seed).
